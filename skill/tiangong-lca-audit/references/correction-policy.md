# 纠偏沉淀政策

人工纠偏是本 Skill 的学习来源，但单次纠偏不会自动成为通用规则。

## 1. 纠偏记录

记录：

- 原判断及证据。
- 人工修正后的判断。
- 修正理由。
- 适用范围和反例。
- 对结论的影响。

使用 `assets/correction-record-template.md`。

## 2. 沉淀去向

| 情况 | 处理 |
| --- | --- |
| 单一案件特殊事实 | 只留案件记录 |
| 可复现的漏判或误判 | 加入 `tests/evals/` |
| 可直接验证的通用逻辑 | 更新 `rules/*.json` 并加测试 |
| 审核顺序或判断边界问题 | 更新对应 reference |
| 平台字段或 API 变化 | 更新输入契约或平台集成代码 |

## 3. 升级门槛

修改通用规则前必须回答：

1. 哪类数据集适用？
2. 哪些证据足以触发？
3. 哪些情况看似相同但不应触发？
4. 默认严重程度是什么？
5. 如何通过回归案例证明没有变得过严？

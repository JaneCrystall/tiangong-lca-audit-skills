# 模型数据集审核方法

模型审核不是“模型表单审核”，而是对目标、结构和关键关联过程的组合审核。

## 1. 模型自身

确认：

- 模型名称、目标产品、功能单位或目标量清楚且一致。
- 模型分类、一般性说明和预期用途与目标一致。
- 方法、边界、时间、地域和技术范围有足够说明。
- 版本、标识和管理信息可追溯。

## 2. 模型结构

检查：

- 图中存在明确目标节点或参考输出。
- 从主要输入到目标输出的关键路径连通。
- 不存在可直接验证的关键节点缺失、重复、孤立或错误连接。
- 节点方向、交换量和连接关系能够支撑目标量。

无法获得模型图或连接关系时，不得确认结构完整。

## 3. 关联过程

1. 列出全部关联过程，并标记其在模型中的角色。
2. 识别决定目标结果的关键过程。
3. 对关键过程执行完整过程审核；对非关键过程至少检查对象、参考流和连接合理性。
4. 汇总各过程阻断问题是否传导到模型结论。

关联过程自身通过，不代表模型连接一定正确；关联过程不通过，也需要说明该问题是否实际影响模型。

## 4. 模型结论

- 关键过程存在影响模型结果的阻断问题：模型不通过。
- 关键关联过程缺失或无法审核：信息不足。
- 结构或替代过程合理性需要专家判断：需人工确认。
- 目标、结构、关键过程和整体边界均可验证且无阻断问题：通过。

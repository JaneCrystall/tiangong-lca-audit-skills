# Workflow

## 审核使用流程

1. 使用 `audit` 一次生成标准化数据、程序预检和 Agent 审核任务包；调试时也可分别运行 `normalize` 与 `check-rules`。
2. 将任务包提供给 Agent，执行语义审核。
3. 过程审核按四个窗口和跨窗口关系执行；模型审核递归检查关键关联过程。
4. 合并程序发现和 Agent 发现，按统一政策聚合结论。
5. 人工确认结论后，再执行平台操作。
6. 人工纠偏进入案件记录，并按门槛升级为规则或评测。

## 维护流程

1. 判断变化属于执行流程、审核方法、结构化规则、输出模板、平台集成还是单一案件。
2. 只修改对应的唯一事实来源。
3. 通用规则变化必须增加回归案例。
4. 检查通过样本不会因规则变化被过严驳回。
5. 运行 Skill 自检和全部测试。

## 案件归档

新批次按 `docs/case-storage.md` 使用 `active/`、`archive/` 和根级 `index.jsonl` 管理；旧案件渐进迁移，不一次性搬动。

## 平台化流程

平台接入按“读取 → 标准化 → 审核 → 人工确认 → 写回”演进。审核判断和平台写操作始终保持独立。
